# Drawing Studio Stack And Projects

The drawing studio stack is the support layer for Conjuration practice:
courses, references, tools, supplies, software ideas, medium experiments, and
small project prompts. It should make practice easier to start and easier to
check, without turning resource collection into the project.

The practice pages own the core loops:
[Constructive Drawing Practice](constructive-drawing-practice.md) and
[Visualization And Imagination Training](visualization-and-imagination-training.md).
This page owns the surrounding stack.

## Resource Backlog

Resource notes should be treated as a backlog, not as an obligation to consume
everything before drawing.

Current useful shelves:

- dynamic sketching;
- Ctrl+Paint;
- ArtWod;
- figure-drawing courses;
- medium-based classes, especially pastel, colored pencil, ink, watercolor, or
  acrylic;
- art reference collections and online video archives;
- settei and design-reference sites for production art, props, and character
  sheets;
- manga craft books and panel studies;
- small concept-art prompts such as head, hand, prop, or creature design.

The rule is to pull one resource into one exercise. A course is useful when it
provides the next constraint, check, or reference source. Otherwise it remains a
bookmark.

## Tool Stack

Digital tools should tighten feedback loops:

- layer-based overdraw for reference checkpoints;
- hidden-reference overlays for proportion checking;
- perspective grids and simple camera setups;
- canvas rotation and vertex hiding for form drills;
- simple 3D scenes for viewpoint prediction;
- 3D sketching apps when they let a line or form be inspected in space.

Potential tool ideas:

- ellipse trainer: draw an ellipse for a given axis pair or degree, then score
  accuracy;
- cube rotation trainer: rotate a cube, hide vertices, and predict the missing
  structure;
- web canvas for quick perspective drills;
- prompt generator that turns abstract words into construction targets.

These are Genesis candidates only when they become buildable software projects.
Until then, they are practice aids.

## Materials And Media

The current physical stack is small and enough to start:

- ballpoint pens for casual permanent marks and light guidelines;
- colored ballpoint pens;
- dip pen and white ink;
- watercolor set;
- pencils when accurate block-in matters more than commitment;
- acrylic, color pencil, pastel, and brush media as medium experiments.

Medium experiments should have a clear purpose. Ink and ballpoint are good for
line confidence and casual volume. Pencil is good for block-in and correction.
Watercolor, acrylic, and color pencil are useful when the target is color
mixing, brush handling, edge softness, or expressive distortion.

The residual art notes add one specific material preference: soft media that
allow very light buildup under normal pressure and require deliberate pressure
for dark marks. Candidates include woodless graphite, soft 2B or 4B leads,
graphite powder, carbon pencils, Pierre Noire, and neutral colored pencils.
This belongs in testing only if the medium supports the current drawing loop
instead of becoming shopping research.

## Design And Painting Projects

Some sources point beyond drawing fundamentals into design and painting. Keep
these as project prompts:

- shape carving;
- totem heads;
- mechtober or Inktober variants;
- fantasy design and settei studies;
- Van Gogh portrait study;
- wooden puzzle painting;
- wallet, sneaker, or object customization;
- album-cover studies;
- expressionist distortion effects;
- suggestion and evocation rather than literal depiction.

These projects should still declare their training target. A painting project
can be about edge softness, color mixing, brush stroke control, stylized form,
or evocative simplification. If it is only "make something nice," it belongs in
casual play, not the main improvement loop.

## Reference Strategy

Use references that make a specific drawing question easier to test:

- multiple views for construction;
- plans and interiors for perspective;
- production sheets for design consistency;
- 3D models for camera prediction;
- museum objects for silhouette, material, and named detail;
- photo sets for lighting and shape language;
- maps or street-view references for environment practice.

When a reference source is too broad, convert it into one prompt:

```text
source -> subject -> constraint -> check
```

For example:

```text
interior photo -> sofa and window corner -> two-point block-in -> compare
vanishing lines
```

or:

```text
settei sheet -> prop silhouette -> rotate into new view -> compare with model
logic
```

## Open Questions

- Which resource should become the first structured drawing sprint?
- Should the ellipse or cube trainer become a small Genesis project?
- Which medium experiment supports the current drawing loop instead of
  distracting from it?
- What reference collection best supports scene-embedded construction?
