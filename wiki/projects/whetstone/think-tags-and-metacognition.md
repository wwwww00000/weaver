# Think Tags And Metacognition

Think tags are short process labels for what the mind is doing. They are not
content notes. A useful tag names the current process just enough to route the
next move: planning, doing, looping, judging, drifting, avoiding, worrying,
searching, or stuck.

The skill is not continuous self-narration. It is a small intervention:
notice, tag, choose a micro-move, and return to the task.

## Controller Loop

Whetstone should treat metacognition as a controller loop:

1. Frame the task.
2. Generate candidate moves.
3. Select one move.
4. Execute for a short interval.
5. Monitor evidence.
6. Communicate or compress the result.

The loop matters because raw effort often keeps optimizing the wrong frame. A
metacognitive controller asks whether the next 15 minutes should be spent
computing, reframing, searching for counterexamples, simplifying the toy case,
or stopping to recover.

## Micro-Tagging Pattern

Good tags are small, nonjudgmental, and paired with a control move.

- `plan`: choose the next concrete action.
- `do`: execute without reopening the frame.
- `stuck`: switch representation or ask for a smaller case.
- `loop`: interrupt rumination or repeated checking.
- `judge`: notice evaluation pressure and return to evidence.
- `worry`: distinguish threat from actionable problem.
- `urge`: surf the impulse before acting.
- `drift`: re-anchor attention.

The tag is a handle, not an essay. During deep work, tagging should happen at
transitions, checkpoints, and stuck points rather than on every thought.

## Problem-Solving Operators

Think tags become more useful when they point to specific operators:

- re-represent the problem;
- test boundaries and extremes;
- search for invariants;
- use symmetry or duality;
- factor the problem into smaller pieces;
- hunt for a counterexample;
- build a toy model;
- ablate one assumption or feature.

This gives the controller something better to do than "try harder." A stuck
tag can route directly to one operator and a short execution window.

## Practice Cadence

A minimal Whetstone cadence can stay small:

- one or two formal noting reps to make tags available;
- a start-of-dive frame before hard work;
- low-friction checkpoints during longer sessions;
- a two-minute postmortem after a meaningful attempt;
- a weekly review of recurring tags and stuck patterns.

Communication practice belongs here too. "Tree first, then fill" is a useful
compression rule: state the claim, give three branches, then add one support
for each branch.

## Relationship To Chronicle

[Chronicle](../chronicle.md) uses think tags for mindfulness, rumination, urge
surfing, and state routing. Whetstone uses them as cognitive-control handles.
The same tag can serve both projects when it preserves awareness and improves
the next move.

## Open Questions

- Which tag set is small enough to remember under stress?
- How should tags differ between meditation, writing, coding, math, and trading
  research?
- What review artifact best preserves metacognitive learning: a log, a weekly
  table, or short wiki notes?
- When does tagging improve control, and when does it interrupt useful flow?
