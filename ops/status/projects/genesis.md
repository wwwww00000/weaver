# Genesis Status

## Current Focus

Building things, exercising agency, AI-adjacent work, and AI research. Current
center is Weaver as a knowledge system, agentic tooling, small AI research
loops, and the build-practice layer that turns ideas into artifacts.

## Recent Changes

- Genesis was synthesized into a project hub with pages for Weaver, agentic
  tooling, AI research, reasoning/tool use, credit assignment, and agency.
- The initial Weaver ingest and synthesis pass is complete.
- Project status cards are being introduced as the live state layer.

## Decisions

- Treat Weaver as both infrastructure and a Genesis project, but keep
  deterministic tooling separate from agentic synthesis.
- Keep AI research hobby-scale and artifact-oriented.
- Use markdown, git diffs, and task templates as the repeatability layer before
  introducing heavier orchestration.

## Next Actions

- Decide how status cards, weekly notes, and project dashboards should fit the
  Obsidian workflow.
- Evaluate whether the next Genesis build should be a context compiler,
  dashboard assembler, notebook controller, command palette, or incremental
  triage tool.
- Keep design questions explicit before adding automation.

## Open Questions

- How much of Weaver synthesis should stay Codex-thread-driven?
- Should subagents or SDK jobs be introduced now, or only after repeated manual
  review passes expose a stable task shape?
- What frontmatter and semantic markers should project pages and status cards
  standardize?
- What should be the first standalone Genesis artifact after Weaver's initial
  bulk pass?

## Blocked Or Waiting

- Waiting on design decisions about Obsidian integration, weekly notes, status
  card schema, and automation boundaries.

## Promote To Wiki

- Stable workflow decisions.
- Status-card schema changes that should become project documentation.
- Agent synthesis/review patterns that survive repeated use.
- New build artifacts or research experiments.
