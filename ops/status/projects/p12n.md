# P12n Status

## Current Focus

Active crypto trading project with a machine-learning focus. Current center is
low-SNR return forecasting, n-linear and sequence-model-inspired structures,
feature transforms, execution/policy translation, and experiment
infrastructure.

## Recent Changes

- P12n was split into a project hub plus child pages for temporal returns,
  n-linear models, sequence-model analogies, feature transforms, execution and
  policy, and experiment infrastructure.
- Broader reusable methods were separated into Quant topic pages.
- Obelisk lessons were explicitly kept as archive context rather than active
  project state.

## Decisions

- Keep P12n and Quant disjoint: P12n is the active applied trading project;
  Quant is the reusable/archive methods layer.
- Treat forecast quality, execution policy, and infrastructure as separate
  loops that must all survive review.

## Next Actions

- Pick the next concrete modeling or experiment-infrastructure task from the
  active codebase.
- Use P12n pages as project-state context before starting a new trading
  research pass.
- Promote new experiment results only after they include validation, leakage,
  and execution-policy implications.
